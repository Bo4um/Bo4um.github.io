Абстрактные классы и интерфейсы

Хранение данных о времени

Описать класс Time, хранящий время в пределах одних суток. В классе должно быть описано одно поле value, являющееся целым числом (int), которое хранит количество минут, прошедших от начала суток. Также в классе должны быть описаны конструктор:

    Time(int hours, int minutes)

При этом конструктор должен проверять условия 0 ≤ hours < 240 и 0 ≤ minutes < 60. Если эти условия не выполняются, конструктор должен генерировать исключительную ситуацию IllegalArgumentException.

Также должны быть описаны методы:

    public int getHours()
    public int getMinutes()
    public Time add(Time time)
    public Time sub(Time time)
    public boolean before(Time time)
    public boolean after(Time time)
    public static Time diff(Time begin, Time end)
    public String toString()

При этом метод toString() должен возвращать время в формате HH:mm.

Хранение данных о событии

Также описать интерфейс для различных событий в календаре Event. В этом интерфейсе объявить методы для:

  • получения строкового названия события
  • получения времени начала события
  • получения времени окончания события

Описать классы, реализующие интерфейс Event, для следующих типов событий:

  1. Учебная пара. Этот класс должен хранить номер пары, название предмета, тип занятия (лекция, практика, лабораторная работа).

    2. При получении строкового названия события должна формироваться строка, состоящая из типа занятия и названия предмета, разделённые запятой.

    При получении времени начала события, определять время по номеру пары:

    • 1-ая пара начинается в 8:00
    • 2-ая пара в 9:35
    • 3-ая пара в 11:30
    • 4-ая пара в 13:05
    • 5-ая пара в 14:40
    • 6-ая пара в 16:35
    • 7-ая пара в 18:10
    • 8-ая пара в 19:45

    При получении времени окончания события, определять время исходя из длительности пары 1 час 25 минут.

  2. Экзамен. Этот класс должен хранить название предмета, количество студентов.

    3. При получении строкового названия события должна формироваться строка, состоящая из слова "экзамен" и названия предмета, разделённых запятой, после названия предмета в круглых скобках должно указываться количество студентов.

    При получении время начала события метод всегда должен возвращать время 8:00.

    При получении времени окончания события, определять время исходя из длительности экзамена, которая рассчитывается по нормативу 30 минут на одного студента.

  3. Зачёт. Этот класс должен хранить номер пары - начала зачёта, название предмета, количество студентов.

    4. При получении строкового названия события должна формироваться строка, состоящая из слова "зачёт" и названия предмета, разделённых запятой, после названия предмета в круглых скобках должно указываться количество студентов.

    При получении времени начала события, определять время по номеру пары (см. требования для события Учебная пара).

    При получении времени окончания события, определять время исходя из длительности зачёта, которая рассчитывается по нормативу 15 минут на одного студента.

  4. Консультация. Этот класс должен хранить время начала и время окончания консультации.

    5. При получении строкового названия события должна просто возвращаться строка "консультация".

    При получении времени начала или окончания события должно возвращаться значение соответствующего поля.

Формирование расписания на день

Используя ввод с клавиатуры, сформировать расписание (массив объектов, реализующих интерфейс Event). Вывести все расписание, используя формат вывода:

<время начала> - <время окончания> <название события>

Дополнительно реализовать

  1. При вводе данных о событиях сделать проверку корректности вводимых пользователем значений:
    • Номер пары должен быть целым числом от 1 до 8.
    • Тип занятия должно быть только значением из представленного списка (как альтернатива - можно вывести пронумерованный список возможных значений и дать пользователю возможность выбора номера пункта из списка с проверкой правильности введённого номера).
    • Количество студентов на экзамене должно быть целым числом от 1 до 16.
    • Количество студентов на зачёте должно быть целым числом от 1 до 32.
    • При вводе времени начала и окончания консультации время начала должно быть строго меньше времени окончания.
  2. Реализовать сортировку событий по времени начала события (пользователь может вводить события не в хронологическом порядке).
  3. Реализовать проверку, чтобы события не пересекались. Для этого отсортированный массив событий нужно просмотреть в цикле начиная со второго и проверить, чтобы время начала события было после времени окончания предыдущего события.
  4. При преобразовании времени в строковое значение реализовать вывод количества часов и количества минут строго с двумя цифрами. Например, время 9 часов 7 минут необходимо выводить как "09:07". После этого формат вывода расписания можно поменять на следующий (расписание должно выводиться в табличном виде):
    5. <время начала> | <время окончания> | <название события>
  5. При выборе пользователем типа события (учебная пара, экзамен, зачёт или консультация) осуществлять выбор объекта, реализующего ещё один интерфейс, который необходимо дополнительно описать:
    6. public interface EventConsoleReader {
    Event readFromConsole(Scaner console);
}
