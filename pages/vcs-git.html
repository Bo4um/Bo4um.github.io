Система контроля версий Git

Написать программу, рассчитывающую стоимость компьютера.

Порядок выполнения задания

  1. Создать репозиторий (проект) в системе GitLab, GitHub или другой подобной.
  2. Скопировать SSH-URL созданного пустого репозитория (проекта) в буфер обмена.
  3. С помощью консольного клиента Git Bash выполнить клонирование репозитория с помощью команды:
    4.                 git clone
  4. Создать Java-проект в IntelliJ IDEA, указав в качестве названия проекта (Name) имя репозитория (проекта), указанного в системе контроля версий, а в качестве папки размещения проекта (Location) указав путь к папке, в которой размещается локальная копия репозитория (проекта).
  5. Описать класс Ram (оперативная память) с полями volume (объём в Гб), clockFrequency (тактовая частота в МГц), cost (стоимость в копейках), а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  6. Описать класс Cpu (процессор) с полями cores (количество ядер), clockFrequency (тактовая частота в ГГц), box (логическое поле, поставляется ли в коробочной версии с системой охлаждения), cost (стоимость в копейках), а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  7. Описать класс Gpu (видеокарта) с полями ramVolume (объём видеопамяти в Гб), clockFrequency (тактовая частота в МГц), shaderProcessors (количество шейдерных потоковых процессоров), cost (стоимость в копейках), а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  8. Описать класс MotherBoard (материнская плата) с полями busClockFrequency (тактовая частота шины в МГц), ramSlots (количество слотов оперативной памяти), cost (стоимость в копейках), а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  9. Описать класс Hdd (жёсткий диск) с полями volume (объём в Гб), ssd (логическое поле, является ли SSD), cost (стоимость в копейках), а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  10. Описать класс SystemUnit (системный блок) с полями ram, cpu, gpu, motherBoard, hdd, являющимися ссылками на объекты соответствующих классов, полем cost (стоимость в копейках), а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  11. Добавить в класс SystemUnit метод totalCost(), который должен посчитать стоимость системного блока вместе со стоимостью всех его комплектующих. Учесть, что некоторые комплектующие могут быть не заданы (соответствующие поля установлены в null). Добавить вывод общей стоимости в метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  12. Описать класс Monitor (монитор) с полями diagonal (размер диагонали в дюймах), width, height (количество пикселей по ширине и высоте), cost (стоимость в копейках), а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  13. Описать класс Keyboard (клавиатура) с полями wireless (логическое поле, является ли беспроводной), lowProfile (логическое поле, является ли низкопрофильной), cost (стоимость в копейках), а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  14. Описать класс Mouse (мышь) с полями wireless (логическое поле, является ли беспроводной), cost (стоимость в копейках), а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  15. Описать класс PersonalComputer (персональный компьютер) с полями systemUnit, monitor, keyboard, mouse, являющимися ссылками на объекты соответствующих классов, а также get-теры и set-теры для этих полей. Сгенерировать метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  16. Добавить в класс PersonalComputer метод totalCost(), который должен посчитать стоимость персонального компьютера вместе со стоимостью всех его комплектующих. Учесть, что некоторые комплектующие могут быть не заданы (соответствующие поля установлены в null). Добавить вывод общей стоимости в метод toString(). Выполнить, используя IDE, операцию Commit and Push.
  17. Описать класс Main с методом main(), в котором реализовать ввод с клавиатуры данных о компьютере и выводе информации о нём.
